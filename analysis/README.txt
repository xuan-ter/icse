================================================================================
                            实验分析说明文档 (README)
================================================================================
本目录包含关于“MIR 优化与 LLVM 优化交互效应”实验的完整分析流程、代码及结果图表。

实验核心目标：
探究 Rust 编译器中 MIR 层优化（Mid-level IR）与 LLVM 层优化（Low-level IR）之间是否存在相互影响（Interaction Effect）。具体来说，我们想知道：开启或关闭某个 MIR 优化，是否会改变某个 LLVM 优化对最终性能的贡献？

2. 实验方法论 (Methodology)
================================================================================
为了严谨地量化 MIR 和 LLVM 优化 Pass 之间的交互效应，我们设计了一个基于统计学的分析框架。以下是详细的方法论说明。

--------------------------------------------------------------------------------
A. 2x2 因子实验设计 (Factorial Design)
--------------------------------------------------------------------------------
对于每一对 MIR Pass $m$ 和 LLVM Pass $l$，我们通过禁用（Ablation）的方式构建了四种实验配置：

1.  **Baseline ($y_{00}$)**: 
    - 配置：MIR=$m$ 开启, LLVM=$l$ 开启 (即默认全开状态)
    - 含义：标准编译流程下的性能。
2.  **MIR Disabled ($y_{10}$)**:
    - 配置：MIR=$m$ 关闭, LLVM=$l$ 开启
    - 含义：单独禁用 MIR Pass 后的性能。
3.  **LLVM Disabled ($y_{01}$)**:
    - 配置：MIR=$m$ 开启, LLVM=$l$ 关闭
    - 含义：单独禁用 LLVM Pass 后的性能。
4.  **Both Disabled ($y_{11}$)**:
    - 配置：MIR=$m$ 关闭, LLVM=$l$ 关闭
    - 含义：两者同时禁用后的性能。

注：所有实验数据均基于多次重复运行（RunID），以消除随机噪声。

--------------------------------------------------------------------------------
B. 数据预处理与对数变换 (Log Transformation)
--------------------------------------------------------------------------------
由于编译器优化的性能改进通常是**乘性**的（例如：优化 A 提速 10%，优化 B 提速 20%，叠加后是 $1.1 \times 1.2 = 1.32$ 倍），直接使用原始运行时间进行加减运算会导致误差。

因此，我们首先对运行时间 $T$ 进行自然对数变换：
$$ z = \ln(T) $$
这使得乘性效应转换为加性效应，便于使用线性模型（如 ANOVA 或 DiD）进行分析。

--------------------------------------------------------------------------------
C. 交互效应模型 (Difference-in-Differences, DiD)
--------------------------------------------------------------------------------
我们定义交互效应 $\Delta$ 为：在“LLVM Pass 关闭”与“LLVM Pass 开启”两种环境下，禁用 MIR Pass 带来的性能变化差异。

数学公式为：
$$ \Delta = (z_{01} - z_{11}) - (z_{00} - z_{10}) $$

- **$z_{00} - z_{10}$**: 在 LLVM 开启时，MIR Pass 的贡献。
- **$z_{01} - z_{11}$**: 在 LLVM 关闭时，MIR Pass 的贡献。

**判定标准**：
- **$\Delta \approx 0$**: **独立 (Independent)**。MIR Pass 的贡献不依赖于 LLVM Pass 是否存在。
- **$\Delta < 0$**: **负交互 (Negative/Redundant)**。
    - 解释：当 LLVM 关闭时，MIR 的贡献比 LLVM 开启时更大。
    - 物理意义：两者功能重叠。当 LLVM 存在时，MIR 的工作被 LLVM 掩盖或替代了；只有当 LLVM 缺席时，MIR 才显现出巨大价值。
- **$\Delta > 0$**: **正交互 (Positive/Interference)**。
    - 解释：当 LLVM 关闭时，MIR 的贡献比 LLVM 开启时更小（甚至为负）。
    - 物理意义：两者相互促进或存在冲突。可能 MIR 为 LLVM 创造了机会（Enabling），或者 MIR 破坏了 LLVM 的优化模式（Disabling）。

--------------------------------------------------------------------------------
D. 统计检验与校正 (Statistical Rigor)
--------------------------------------------------------------------------------
为了确保结论的可靠性，我们引入了严格的统计检验流程：

1.  **Bootstrap 重采样 (Bootstrap Resampling)**:
    - 原因：运行时间数据通常不服从正态分布，且样本量有限。
    - 方法：对每个配置 ($y_{00}, y_{10}, y_{01}, y_{11}$) 进行 1000 次有放回重采样，计算出 1000 个 $\Delta$ 值，形成经验分布。
    - 产出：95% 置信区间 (Confidence Interval, CI) = [2.5百分位, 97.5百分位]。

2.  **假设检验 (Hypothesis Testing)**:
    - 零假设 ($H_0$): $\Delta = 0$ (无交互)。
    - P-value 计算：基于 Bootstrap 分布计算 $\Delta$ 显著异于 0 的概率。

3.  **多重假设检验校正 (FDR Correction)**:
    - 问题：我们需要测试数百个 Pass 对。如果直接用 P < 0.05，会产生大量假阳性（Type I Error）。
    - 解决方案：使用 **Benjamini–Hochberg** 方法控制错误发现率 (False Discovery Rate, FDR)。
    - 标准：校正后的 P值 ($P_{adj}$) < 0.05 被认为具有统计显著性。

================================================================================
3. 关键脚本说明
================================================================================

(1) `analyze_interaction.py`
    - **作用**：数据清洗、对数变换、DiD 计算、FDR 校正。
    - **输出**：生成 `interaction_results.csv`，这是后续所有绘图的基础。

(2) `classify_and_plot.py`
    - **作用**：将交互效应分为三类（负交互、正交互、独立），并分别生成 CSV 和森林图。
    - **输出**：`classified_results/` 目录下的所有内容。

(3) `plot_coupling.py`
    - **作用**：生成复杂的热力图和网络图，用于揭示 Pass 之间的结构化关系。
    - **输出**：`coupling_plots/` 中的热力图和二部图。

(4) `plot_knowledge_graph.py`
    - **作用**：生成力导向布局的知识图谱，展示编译器的拓扑结构。
    - **输出**：`coupling_plots/coupling_knowledge_graph.png`。

4. 图表意义解读 (Interpretation)
================================================================================

--------------------------------------------------------------------------------
A. 基础统计图
--------------------------------------------------------------------------------
1. **`top_interactions_forest.png` (森林图)**
   - **内容**：展示了交互效应最显著的前 20 对 Pass。
   - **解读**：圆点代表交互强度，横线代表置信区间。如果不与 0 线相交，说明交互显著。
   - **用途**：论文中用于展示最核心的定量发现。

2. **`interaction_heatmap.png` (全局热力图)**
   - **内容**：展示所有 MIR Pass (Y轴) 和 LLVM Pass (X轴) 的交互矩阵。
   - **解读**：深色区域代表强交互。可以直观看到哪些区域“很热闹”，哪些区域“很冷清”。

--------------------------------------------------------------------------------
B. 进阶可视化 (`coupling_plots/` 目录)
--------------------------------------------------------------------------------
3. **`filtered_heatmap.png` (纯净热力图)**
   - **内容**：仅保留统计显著 (Significant=True) 的格子，其余留白。
   - **意义**：去伪存真，展示真实的交互分布。证明了大部分优化是独立的 (稀疏性)，同时突出了关键的耦合点。

4. **`clustered_heatmap.png` (聚类热力图)**
   - **内容**：带树状图 (Dendrogram) 的热力图。
   - **意义**：
     - **树状线**：展示了 Pass 之间的功能相似性。如果两个 MIR Pass 被聚在一起，说明它们对下游 LLVM 的影响模式一致。
     - **色块聚集**：揭示了“优化功能群”。

5. **`coupling_network_top50.png` (二部图)**
   - **内容**：左侧 MIR 节点，右侧 LLVM 节点，连线代表强交互。
   - **颜色**：蓝色 = 负交互 (冗余/备份)；红色 = 正交互 (干扰/冲突)。
   - **意义**：直观展示跨层耦合。

6. **`coupling_knowledge_graph.png` (知识图谱)**
   - **内容**：力导向布局图。方块=MIR，圆形=LLVM。
   - **节点大小**：代表“连接度” (Degree)，即该 Pass 的交互影响力。
   - **布局**：处于中心的节点是编译器的“枢纽” (Hubs)，如 `Inline`。边缘节点则是相对独立的。
   - **意义**：展示编译器的优化拓扑结构，揭示谁是核心，谁是配角。

================================================================================
5. 结果分类说明 (`classified_results/` 目录)
================================================================================
- **Negative Interaction (负交互, Delta < 0)**:
  - 含义：冗余或备份关系。两个 Pass 功能重叠。
  - 典型例子：MIR Inline + LLVM LICM。MIR 做了内联，LLVM 就不用做了；MIR 没做，LLVM 帮忙兜底。

- **Positive Interaction (正交互, Delta > 0)**:
  - 含义：干扰或竞争关系。
  - 典型例子：MIR 的某个简化破坏了 LLVM 的模式匹配，导致总收益下降。

- **Independent (独立)**:
  - 含义：互不干扰，收益可叠加。大多数 Pass 属于此类。
