## 6. 第二步：运行项目级 DID 分析

### 6.1 标准分析目录

每个项目应使用如下两类目录之一：

- `<project>\analysis\did\`
- `<project>\analysis_new\did\`

标准脚本通常包括：

- `analyze_interaction.py`
- `classify_and_plot.py`
- `plot_coupling.py`
- `plot_knowledge_graph.py`

### 6.2 当前 DID 数学口径

当前分析先对运行时间取对数：

```text
z = log(runtime)
```

然后使用以下 DiD 公式：

```text
delta = (mean(z01) - mean(z11)) - (mean(z00) - mean(z10))
```

其中：

- `y00`：基线
- `y10`：仅 MIR 改变
- `y01`：仅 LLVM 改变
- `y11`：MIR 和 LLVM 同时改变

### 6.3 当前显著性规则

当前代码里，一个交互对只有同时满足下面两个条件才算显著：

```text
BH-adjusted p-value < 0.05
且
95% bootstrap CI excludes 0
```

### 6.4 当前 bootstrap 设置

当前项目脚本统一采用：

```text
BOOTSTRAP_SAMPLES = 2000
BOOTSTRAP_SEED = 42
```

## 7. 基线处理

### 7.1 为什么要单独说明

DID 分析要求存在一个分析基线 `(None, None)`，也就是：

- `MIR_Pass=None`
- `LLVM_Pass=None`

但部分项目的原始 CSV 并不是这样编码基线，而是把“全关闭基线”写成：

- `LLVM_Pass=None`
- `MIR_Pass=All`

### 7.2 当前正确处理方式

对于这种数据，当前正确复现方式是：

1. 读取原始结果
2. 把 `(MIR_Pass=All, LLVM_Pass=None)` 映射成分析口径里的 `(None, None)`
3. 再构造 `y00/y10/y01/y11`

当前至少以下项目需要注意这一点：

- `rustls`
- `quinn`
- `aho-corasick`

如果不做这一步，常见问题有两类：

- 找不到 `y00`
- 看似能跑通，但实际用了错误基线

## 11. 常见坑

### 11.1 `analyze_interaction.py` 指向旧 results 目录

很多项目的 `analyze_interaction.py` 里有固定的 `DATA_DIR`。重跑前先确认它是否指向你要分析的那次结果。

### 11.2 基线编码和分析口径不一致

不要默认所有项目都直接有 `(None, None)` 基线。对于 `rustls`、`quinn`、`aho-corasick` 这类项目，通常需要先做 `(All, None) -> (None, None)` 的归一化。

### 11.3 Linux 路径残留

有些旧脚本最初是在 Linux 环境写的，可能仍带有 `/root/...` 路径。Windows 下直接运行前要先改。

### 11.4 “没有显著交互”不等于“没有强信号”

某些项目可能出现：

- `delta` 很大
- `CI` 不跨 0
- 但 `p_adj >= 0.05`

原因通常是：

- 检验数很多
- BH 校正很严格
- bootstrap `p_value` 的分辨率有限

`rustls` 当前就是这种典型情况。

## 15. 一段话版本

当前实验的完整复现方式是：先运行各项目的 `experiment_*` 脚本，在 `results\<时间戳>` 下生成原始运行数据；再进入该项目的 `analysis\did` 或 `analysis_new\did`，用 `analyze_interaction.py` 基于 `log(runtime)` 执行 bootstrap DiD，并按 `BH-adjusted p < 0.05` 且 `95% bootstrap CI excludes 0` 的规则判定显著交互；随后运行分类图、耦合图和知识图谱脚本；最后执行 `datas\interaction_stats\summarize_interactions.py` 生成跨项目总表。对于 `rustls`、`quinn`、`aho-corasick` 这类原始基线编码为 `LLVM_Pass=None, MIR_Pass=All` 的项目，必须先归一化成分析口径 `(None, None)`，否则 DID 基线会不一致。
